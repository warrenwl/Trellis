# 思维指南

> **目的**：扩展您的思维以发现您可能未曾考虑到的事情。

---

## 为什么要有思维指南？

**大多数 bug 和技术债务来自"没想到"**，而非技能不足：

- 没想到层边界会发生什么 → 跨层 bug
- 没想到代码模式重复 → 到处重复的代码
- 没想到边缘情况 → 运行时错误
- 没想到未来的维护者 → 无法阅读的代码

这些指南帮助您在编码前**提出正确的问题**。

## 快速参考：思维触发器

### 何时思考跨层问题

- [ ] 功能涉及 3+ 层（API、Service、Component、Database）
- [ ] 数据格式在层之间变化
- [ ] 多个消费者需要相同数据
- [ ] 您不确定把某些逻辑放在哪里

→ 阅读[跨层思维指南](./cross-layer-thinking-guide.md)

### 何时思考代码复用

- [ ] 您正在编写与已存在代码类似的代码
- [ ] 您看到相同模式重复 3+ 次
- [ ] 您在多个地方添加新字段
- [ ] **您正在修改任何常量或配置**
- [ ] **您正在创建新的工具/辅助函数** ← 先搜索！

→ 阅读[代码复用思维指南](./code-reuse-thinking-guide.md)

### 何时思考跨平台问题

- [ ] 编写用户将直接运行的脚本
- [ ] 添加使用示例或帮助文本
- [ ] 处理文件路径或命令
- [ ] **从 shell 脚本迁移到 Python**

→ 阅读[跨平台思维指南](./cross-platform-thinking-guide.md)

## 修改前规则（关键）

> **在更改任何值之前，始终先搜索！**

```bash
# 搜索您要更改的值
grep -r "value_to_change" .
```

这一个习惯可以防止大多数"忘记更新 X"的 bug。

## 如何使用本目录

1. **编码前**：浏览相关的思维指南
2. **编码时**：如果感觉重复或复杂，检查指南
3. **bug 后**：将新见解添加到相关指南中（从错误中学习）
