# 工作区索引

> 所有开发者 AI 代理工作记录的目录。

---

## 概述

本目录追踪在本项目中与 AI 代理工作的所有开发者的记录。

### 文件结构

```
workspace/
|-- index.md              # 本文件 - 主索引
+-- {developer}/          # 每个开发者目录
    |-- index.md          # 带会话历史的个人索引
    |-- tasks/         # 任务文件
    |   |-- *.json        # 活跃任务
    |   +-- archive/      # 按月归档的任务
    +-- journal-N.md     # 日志文件（顺序：1, 2, 3...）
```

## 入门

### 新开发者

运行初始化脚本：

```bash
python3 ./.trellis/scripts/init_developer.py <your-name>
```

这将：
1. 创建您的身份文件（被 gitignore）
2. 创建您的进度目录
3. 创建您的个人索引
4. 创建初始日志文件

### 回归的开发者

1. 获取您的开发者名称：
   ```bash
   python3 ./.trellis/scripts/get_developer.py
   ```

2. 读取您的个人索引：
   ```bash
   cat .trellis/workspace/$(python3 ./.trellis/scripts/get_developer.py)/index.md
   ```

## 指南

### 日志文件规则

- 每个日志文件**最多 2000 行**
- 达到限制时，创建 `journal-{N+1}.md`
- 创建新文件时更新您的个人 `index.md`

### 会话记录格式

每个会话应包括：
- 摘要：一行描述
- 主要更改：修改了什么
- Git 提交：提交哈希和消息
- 下一步：接下来要做什么

---

## 会话模板

记录会话时使用此模板：

```markdown
## 会话 {N}: {标题}

**日期**: YYYY-MM-DD
**任务**: {task-name}

### 摘要

{一行摘要}

### 主要更改

- {更改 1}
- {更改 2}

### Git 提交

| 哈希 | 消息 |
|------|---------|
| `abc1234` | {提交消息} |

### 测试

- [OK] {测试结果}

### 状态

[OK] **已完成** / # **进行中** / [P] **受阻**

### 下一步

- {下一步 1}
- {下一步 2}
```

---

**语言**：所有文档必须用**英文**编写。
