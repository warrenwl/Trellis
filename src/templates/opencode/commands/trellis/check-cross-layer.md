# 跨层检查

检查你的更改是否考虑了所有维度。大多数 bug 来自"没想到"，而非技术能力不足。

> **注意**：这是**实现后**的安全网。理想情况下，在编写代码之前阅读[实现前检查清单](.trellis/spec/guides/pre-implementation-checklist.md)。

## 执行步骤

### 1. 识别更改范围

```bash
git status
git diff --name-only
```

### 2. 选择适用的检查维度

根据你的更改类型，执行以下相关检查：

---

## 维度 A：跨层数据流（涉及 3 层或以上时必需）

**触发条件**：更改涉及 3 层或以上

| 层 | 常见位置 |
|-------|------------------|
| API/路由 | `routes/`, `api/`, `handlers/`, `controllers/` |
| 服务/业务逻辑 | `services/`, `lib/`, `core/`, `domain/` |
| 数据库/存储 | `db/`, `models/`, `repositories/`, `schema/` |
| UI/展示 | `components/`, `views/`, `templates/`, `pages/` |
| 工具 | `utils/`, `helpers/`, `common/` |

**检查清单**：
- [ ] 读取流程：数据库 → 服务 → API → UI
- [ ] 写入流程：UI → API → 服务 → 数据库
- [ ] 类型/模式在各层之间正确传递？
- [ ] 错误正确传播给调用者？
- [ ] 各层处理了加载/待处理状态？

**详细指南**：`.trellis/spec/guides/cross-layer-thinking-guide.md`

---

## 维度 B：代码重用（修改常量/配置时必需）

**触发条件**：
- 修改 UI 常量（标签、图标、颜色）
- 修改任何硬编码值
- 在多个地方看到类似代码
- 创建新的工具/辅助函数
- 刚完成跨文件的批量修改

**检查清单**：
- [ ] 先搜索：这个值定义在多少地方？
  ```bash
  # 在源文件中搜索（根据项目调整扩展名）
  grep -r "value-to-change" src/
  ```
- [ ] 如果 2+ 处定义相同值 → 应该提取到共享常量
- [ ] 修改后，所有使用点都更新了？
- [ ] 如果创建工具：类似工具已存在？

**详细指南**：`.trellis/spec/guides/code-reuse-thinking-guide.md`

---

## 维度 B2：新工具函数

**触发条件**：即将创建新的工具/辅助函数

**检查清单**：
- [ ] 先搜索现有类似工具
  ```bash
  grep -r "functionNamePattern" src/
  ```
- [ ] 如果存在类似，能扩展它吗？
- [ ] 如果创建新的，放在正确位置（共享 vs 领域特定）？

---

## 维度 B3：批量修改后

**触发条件**：刚在多个文件中修改了类似模式

**检查清单**：
- [ ] 检查了所有类似模式的文件？
  ```bash
  grep -r "patternYouChanged" src/
  ```
- [ ] 有遗漏应该更新的文件吗？
- [ ] 这个模式应该抽象化以防止未来重复？

---

## 维度 C：导入/依赖路径（创建新文件时必需）

**触发条件**：创建新源文件

**检查清单**：
- [ ] 使用正确的导入路径（相对 vs 绝对）？
- [ ] 没有循环依赖？
- [ ] 与项目的模块组织一致？

---

## 维度 D：同层一致性

**触发条件**：
- 修改显示逻辑或格式化
- 同一领域概念在多处使用

**检查清单**：
- [ ] 搜索使用同一概念的其他地方
  ```bash
  grep -r "ConceptName" src/
  ```
- [ ] 这些用法一致吗？
- [ ] 他们应该共享配置/常量？

---

## 常见问题快速参考

| 问题 | 根因 | 预防 |
|-------|------------|------------|
| 改了一处，遗漏其他 | 没搜索影响范围 | 改之前先 `grep` |
| 数据在某层丢失 | 没检查数据流 | 追踪数据源到目的地 |
| 类型/模式不匹配 | 跨层类型不一致 | 使用共享类型定义 |
| UI/输出不一致 | 同一概念在多处 | 提取共享常量 |
| 类似工具已存在 | 没先搜索 | 创建前先搜索 |
| 批量修复不完整 | 没验证所有出现 | 修复后 grep |
