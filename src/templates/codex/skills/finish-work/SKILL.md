# 完成工作 - 提交前检查清单

在提交或提交之前，使用此检查清单确保工作完整性。

**时机**：代码编写和测试完成后，提交之前

---

## 检查清单

### 1. 代码质量

```bash
# 必须通过
pnpm lint
pnpm type-check
pnpm test
```

- [ ] `pnpm lint` 通过且无错误？
- [ ] `pnpm type-check` 通过且无类型错误？
- [ ] 测试通过？
- [ ] 没有 `console.log` 语句（使用日志器）？
- [ ] 没有非空断言（`x!` 操作符）？
- [ ] 没有 `any` 类型？

### 2. 代码规范同步

**代码规范文档**：
- [ ] `.trellis/spec/backend/` 需要更新吗？
  - 新模式、新模块、新约定
- [ ] `.trellis/spec/frontend/` 需要更新吗？
  - 新组件、新 hooks、新模式
- [ ] `.trellis/spec/guides/` 需要更新吗？
  - 新的跨层流程、bug 教训

**关键问题**：
> "如果我修复了一个 bug 或发现了非显而易见的东西，我是否应该记录下来，这样未来的我（或其他人）就不会遇到同样的问题？"

如果是 → 更新相关的代码规范文档。

### 2.5. 代码规范硬性阻止（基础设施/跨层）

如果此更改涉及基础设施或跨层契约，这是一个阻止性检查清单：

- [ ] 规范内容是可执行的（真实签名/契约），而不仅仅是原则性文本
- [ ] 包含文件路径 + 命令/API 名称 + 有效载荷字段名
- [ ] 包含验证和错误矩阵
- [ ] 包含好/基础/坏情况
- [ ] 包含所需测试和断言点

**阻止规则**：
如果基础设施/跨层已更改但相关规范仍然是抽象的，**不要完成**。先手动运行 `$update-spec`。

### 3. API 更改

如果修改了 API 端点：

- [ ] 输入模式更新了？
- [ ] 输出模式更新了？
- [ ] API 文档更新了？
- [ ] 客户端代码更新以匹配？

### 4. 数据库更改

如果修改了数据库模式：

- [ ] 迁移文件已创建？
- [ ] 模式文件已更新？
- [ ] 相关查询已更新？
- [ ] 种子数据更新了（如果适用）？

### 5. 跨层验证

如果更改跨越多层：

- [ ] 数据在各层之间正确流动？
- [ ] 错误处理在各边界正常工作？
- [ ] 类型在各层之间一致？
- [ ] 处理了加载状态？

### 6. 手动测试

- [ ] 功能在浏览器/应用中工作？
- [ ] 边缘情况已测试？
- [ ] 错误状态已测试？
- [ ] 页面刷新后工作？

---

## 快速检查流程

```bash
# 1. 代码检查
pnpm lint && pnpm type-check

# 2. 查看更改
git status
git diff --name-only

# 3. 根据更改的文件，检查上述相关项目
```

---

## 常见遗漏

| 遗漏 | 后果 | 检查 |
|-----------|-------------|-------|
| 代码规范文档未更新 | 其他人不知道更改 | 检查 .trellis/spec/ |
| 规范文本仅是抽象的 | 基础设施/跨层更改容易回归 | 要求签名/契约/矩阵/案例/测试 |
| 未创建迁移 | 模式不同步 | 检查 db/migrations/ |
| 类型未同步 | 运行时错误 | 检查共享类型 |
| 测试未更新 | 虚假信心 | 运行完整测试套件 |
| 留下 console.log | 生产日志嘈杂 | 搜索 console.log |

## 与其他命令的关系

```
开发流程：
  编写代码 -> 测试 -> $finish-work -> git 提交 -> $record-session
                          |                              |
                   确保完整性                    记录进度

调试流程：
  遇到 bug -> 修复 -> $break-loop -> 知识捕获
                       |
                  深度分析
```

- `$finish-work` - 检查工作完整性（此技能）
- `$record-session` - 记录会话和提交
- `$break-loop` - 调试后的深度分析

---

## 核心原则

> **交付不仅包括代码，还包括文档、验证和知识捕获。**

完成的工作 = 代码 + 文档 + 测试 + 验证
